他のファクタリングアルゴリズムの効率性

前節で紹介した分析結果から、2つの疑問が生まれた：

  • より高速なアルゴリズムはあるのか？
  • 実行時間を厳密に証明できるアルゴリズムはあるのか？

実行時間が厳密に証明できる最も効率的なアルゴリズムは、二次形式を用いたものです。
これは確率的アルゴリズムであり、期待される実行時間はLn[1/2,1+o(1)]\mathrm{L}_{n}[1 / 2,1+o(1)]Ln​[1/2,1+o(1)]です。これは、二次ふるいの実行時間と同じです。

しかし、実際には、二次ふるいの方が効率的です。

楕円曲線法 (ECM) も確率的アルゴリズムの一つです。これは p - 1 法に似ており、期待される実行時間はLp[12,12]L_{p}\left[\frac{1}{2}, \sqrt{\frac{1}{2}}\right]Lp​[21​,21​​]で、pはnの最小素因数です。この点は、二次ふるい法との大きな違いです。

QS の実行時間は主に n の大きさに依存するが、ECM は n の素因数が小さければ高速です。そのため、ECMはn\sqrt{n}n​よりもかなり小さい素因数を見つけるために使われます。

しかし、素因数の大きさが n\sqrt{n}n​の場合、ECM の実行時間はLn[1/2,1]\mathrm{L}_{\mathrm{n}}[1 / 2,1]Ln​[1/2,1]となり、QS の実行時間と同じになるが、実際には ECM のほうが効率が悪いです。

1988年までは、最速の整数の因数分解アルゴリズムの実行時間はLn[1/2,1]\mathrm{L}_{\mathrm{n}}[1 / 2,1]Ln​[1/2,1] でした。もっと速い整数の因数分解アルゴリズムはないと思っていた人もいました。

しかし、1988年にJohn Pollardがnumber field sieve(NFS)を発明しました。適切な仮定の下で、NFSの実行時間はLn[1/3,(64/9)1/3]\mathrm{L}_{\mathrm{n}}\left[1 / 3,(64 / 9)^{1 / 3}\right]Ln​[1/3,(64/9)1/3]であることが示されました。したがって、NFSはQSよりもはるかに多項式時間アルゴリズムに近いものです。

1980年代以降、ファクタリング・アルゴリズムの分野では劇的な進歩が見られました。そのため、いつの日か多項式時間ファクタリングアルゴリズムが発見される可能性は大きいです。
